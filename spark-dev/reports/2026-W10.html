Implementing SaveMode.Overwrite in my v2 DataSource

8 条回复 · 3 位参与者

Joeri van Ruth 正在为 MonetDB 开发自定义 Spark DataSource V2 以提升写入性能，但在实现 SaveMode.Overwrite 时对表生命周期管理感到困惑。讨论澄清了使用 df.write.save() 时无需实现 CatalogPlugin，连接器内部直接处理表创建与覆盖逻辑是正确做法。

@Joeri van Ruth: 标准 JDBC 写入效率低，需通过自定义 DataSource 实现批量加载，但不确定如何正确集成 Overwrite 模式下的表检测与创建机制。

@Mich Talebzadeh: 建议使用 V2 API 并实现 SupportsOverwriteV2；明确指出 df.write.save() 路径不经过 CatalogPlugin，连接器内部自行处理表生命周期（如 TRUNCATE 或 CREATE）是预期模式。

@Khalid Mammadov: 分享了一篇关于使用 V2 API 和定义 Catalog 的博客文章链接，供参考。

Joeri 确认将按照建议，在 V2 API 中实现 SupportsOverwriteV2，并在连接器内部处理表的生命周期逻辑。
