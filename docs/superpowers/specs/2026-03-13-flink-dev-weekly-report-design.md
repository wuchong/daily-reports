# Flink Dev Weekly Report 设计文档

## 概述

自动抓取 Apache Flink dev 邮件列表一周内的讨论内容，按公告、投票、讨论、JIRA 分类整理，生成周报并推送到钉钉和 GitHub Pages。

## 数据源

- **API**: Apache Pony Mail API
- **端点**: `https://lists.apache.org/api/stats.lua?list=dev&domain=flink.apache.org&d=lte=7d`
- **返回数据**: emails[], thread_struct[]

## 分类规则

| 分类 | 识别规则 | 展示方式 |
|------|----------|----------|
| 公告 | `[ANNOUNCE]` 开头 | 只列标题 + 链接 |
| 投票 | `[VOTE]` 或 `[RESULT]` 开头 | 标题 + 链接；+0/-0/-1 换行显示异议内容 |
| 讨论 | `[DISCUSS]` 开头或无特殊标签 | 展开介绍进展、观点、Thread 链接 |
| JIRA | 包含 `[jira]` | 统计数量 + LLM 生成整体摘要 |

分类后去除标签前缀做展示。

## 数据流

```
Pony Mail API → raw_emails.json → (分类/归并) → threads.json
    → (LLM摘要) → summary.json → HTML + 钉钉
```

## 处理流程

### 1. 邮件抓取 (fetch_emails.py)

调用 Pony Mail API 获取一周邮件，输出 `raw_emails.json`:

```json
{
  "week": "2024-W11",
  "date_range": {"start": "2024-03-04", "end": "2024-03-10"},
  "emails": [...],
  "thread_struct": [...]
}
```

### 2. 分类与归并

- 按标签识别分类（优先级：JIRA > 公告 > 投票 > 讨论）
- 使用 API 返回的 `thread_struct` 归并 Thread
- 每个 Thread 持久化链接: `https://lists.apache.org/thread/{mid}`
- 投票异议检测：扫描 `-1`、`-0`、`+0` 关键词

输出 `threads.json`:

```json
{
  "announcements": [{"subject": "...", "link": "..."}],
  "votes": [{"subject": "...", "link": "...", "has_objection": false, "objection_emails": []}],
  "discussions": [{"subject": "...", "link": "...", "reply_count": 12, "participants": [...], "emails": [...]}],
  "jira_count": 23,
  "jira_titles": [...]
}
```

### 3. LLM 摘要生成 (generate_summary.py)

| 场景 | 输入 | 输出 |
|------|------|------|
| 讨论摘要 | Thread 内所有邮件正文 | 进展、主要观点、结论 |
| 投票异议说明 | 含 -1/-0/+0 的邮件正文 | 异议原因简述 |
| JIRA 摘要 | 所有 JIRA 邮件标题 | 2-3 句话整体摘要 |

API 配置复用: `OPENAI_BASE_URL` / `OPENAI_API_KEY`

### 4. 报告生成 (generate_report.py)

生成 HTML 报告到 `docs/flink-dev/reports/{week}.html`，更新 `index.html` 索引。

### 5. 钉钉推送 (send_dingtalk.py)

Markdown 消息格式:

```markdown
## Flink 社区周报 (2024-03-04 ~ 2024-03-10)

📢 **公告** (2)
- Release Apache Flink 1.19.0 [🔗](https://lists.apache.org/thread/xxx)
- Welcome New PMC Member [🔗](https://lists.apache.org/thread/xxx)

🗳️ **投票** (3)
- Release 1.19.0 RC1 → ✅ 已通过 [🔗](https://lists.apache.org/thread/xxx)
- FLIP-456 State TTL 增强 [🔗](https://lists.apache.org/thread/xxx)
  ⚠️ @zhangsan 提出 -1，认为需要更多性能测试数据

💬 **讨论** (2)

**1. Checkpoint 性能优化** [🔗](https://lists.apache.org/thread/xxx)
本周围绕大状态场景下 Checkpoint 超时问题展开讨论。@zhangsan 建议采用增量快照，@lisi 倾向于先优化文件系统写入。社区计划下周进行 POC 验证。

**2. Watermark 对齐策略** [🔗](https://lists.apache.org/thread/xxx)
讨论多源场景下 Watermark 不对齐导致的数据延迟问题。@wangwu 分享了生产案例，建议引入全局协调机制。尚未达成共识。

🎫 **JIRA**: 本周新建 23 个
本周 JIRA 主要集中在 Checkpoint 稳定性改进、Kafka Connector 偏移量处理等方面。

🔗 [查看完整报告](https://xxx.github.io/daily-reports/flink-dev/reports/2024-W11.html)
```

字符限制处理：讨论超过 5 个时只展示 Top 5（按回复数排序）。

## GitHub Actions Workflow

```yaml
name: Flink Dev Weekly Report

on:
  schedule:
    - cron: '0 1 * * 4'  # UTC 01:00 周四 = 北京 09:00
  workflow_dispatch:
    inputs:
      week:
        description: '报告周数 (YYYY-Www), 留空则为上一周'
        required: false
        default: ''

permissions:
  contents: write

jobs:
  generate-report:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'

      - name: Install dependencies
        run: pip install requests openai

      - name: Calculate week
        id: week
        run: |
          if [ -n "${{ github.event.inputs.week }}" ]; then
            echo "week=${{ github.event.inputs.week }}" >> $GITHUB_OUTPUT
          else
            echo "week=$(date -d 'last monday - 7 days' +%G-W%V)" >> $GITHUB_OUTPUT
          fi

      - name: Fetch emails
        env:
          REPORT_WEEK: ${{ steps.week.outputs.week }}
        run: python scripts/flink-dev/fetch_emails.py

      - name: Generate summary with LLM
        env:
          OPENAI_BASE_URL: ${{ secrets.OPENAI_BASE_URL }}
          OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
        run: python scripts/flink-dev/generate_summary.py

      - name: Generate reports
        env:
          GITHUB_PAGES_URL: https://${{ github.repository_owner }}.github.io/${{ github.event.repository.name }}
        run: python scripts/flink-dev/generate_report.py

      - name: Send to DingTalk
        env:
          DINGTALK_WEBHOOK: ${{ secrets.DINGTALK_WEBHOOK }}
          DINGTALK_SECRET: ${{ secrets.DINGTALK_SECRET }}
          GITHUB_PAGES_URL: https://${{ github.repository_owner }}.github.io/${{ github.event.repository.name }}
        run: python scripts/flink-dev/send_dingtalk.py

      - name: Deploy to pages branch
        run: |
          cp -r docs/flink-dev /tmp/flink-dev-output
          mkdir -p /tmp/flink-dev-output/assets
          cp assets/flink-dev/style.css /tmp/flink-dev-output/assets/

          git fetch origin pages || true
          if git show-ref --verify --quiet refs/remotes/origin/pages; then
            git checkout pages
          else
            git checkout --orphan pages
            git rm -rf . || true
          fi

          rm -rf flink-dev || true
          cp -r /tmp/flink-dev-output flink-dev

          # 更新根 index.html 添加 Flink 入口

          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add -A
          git diff --staged --quiet || git commit -m "chore: add flink-dev weekly report for ${{ steps.week.outputs.week }}"
          git push origin pages
```
